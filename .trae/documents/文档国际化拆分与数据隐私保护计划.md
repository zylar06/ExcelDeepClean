# 文档拆分与隐私清理计划

## 1. 文档拆分
为了更好的阅读体验，将单文件双语 `README.md` 拆分为两个独立文件：
- `README.md`: 英文主文档（符合 GitHub 惯例）。
- `README_zh-CN.md`: 中文文档。
- 在两个文件的头部互相添加链接。

## 2. 隐私清理
- **删除 `test_data.xlsx`**：从 git 历史和文件系统中删除。
- **更新测试代码**：
  - 如果 `test_data.xlsx` 被删除，`tests/test_integration.py` 将无法运行。
  - 需要创建一个**合成的/脱敏的** `test_data.xlsx` 生成脚本（使用 `openpyxl` 代码生成一个简单的测试 Excel），或者在测试中 Mock 一个 Excel 文件。
  - 考虑到时间成本，最快的方法是**生成一个包含假数据的 `tests/fixtures/sample.xlsx`**，并在测试中使用它。

## 3. 实施步骤
1.  **拆分文档**：
    - 创建 `README_zh-CN.md`，将中文内容移入。
    - 清理 `README.md`，仅保留英文内容，并添加指向中文版的链接。
2.  **清理数据**：
    - `git rm test_data.xlsx`。
    - 更新 `.gitignore` 以忽略所有 `.xlsx` 文件（除测试 fixture 外）。
3.  **修复测试**：
    - 创建 `tests/create_sample_excel.py` 用于生成测试用的 Excel。
    - 修改 `tests/test_integration.py`，在测试设置阶段动态生成 Excel 文件，测试结束后清理。
4.  **提交更改**：推送到 GitHub。
